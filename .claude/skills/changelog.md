# CHANGELOG 관리 스킬

## 형식
Keep a Changelog (https://keepachangelog.com/en/1.1.0/) 형식 사용.

## 섹션 카테고리
- **Added**: 새로운 기능
- **Changed**: 기존 기능 변경
- **Deprecated**: 곧 제거될 기능
- **Removed**: 제거된 기능
- **Fixed**: 버그 수정
- **Security**: 보안 관련 수정

## 업데이트 타이밍
- 모든 `feat:`, `fix:` 커밋 시 `[Unreleased]` 섹션 업데이트
- 릴리스 시 `[Unreleased]` → `[x.y.z] - YYYY-MM-DD` 변환

## 작성 규칙
- 각 항목은 동사 원형으로 시작 (예: "Add ...", "Fix ...")
- 영문으로 작성
- 관련 이슈/PR 번호 링크 추가 (있는 경우)
